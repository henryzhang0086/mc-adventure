# 🗺 MC 大冒险 — 升级路线图 (Roadmap)

> 基于 **v1.2.0** 现状制定。当前已是「可交付的工业级单文件横版闯关」，基建（固定步长、DPR 渲染、存档、暂停菜单、移动端触控、构建/测试/部署链路）已完善。
> 因此本路线图重心放在 **内容深度 / 视听炫酷度 / 耐玩度**，而非补基础设施。

## 🎯 设计北极星（每次升级都不能违背）
1. **纯前端、零运行时依赖、单文件可玩** — 任何新功能都要能内联进 `MC大冒险-单文件版.html`。
2. **移动优先** — 手机/iPad 上必须好玩、不卡（性能预算：中端手机 60fps）。
3. **炫酷 + 可玩** — 这是项目初衷；每个版本至少推进其一。
4. **不破坏存档** — `localStorage` 结构向后兼容（加版本号迁移）。

## 🚀 版本路线图（建议顺序执行）

### v1.3 「打击感与视听」— 让现有内容立刻更爽 ⭐ 推荐先做
**目标**：不加新关卡，纯靠「juice」把已有玩法的手感和炫酷度拉满。低风险、改动集中、立竿见影。
- **打击感**：命中顿帧(hit-stop)、受击白闪、击退强化、踩踏弹跳更脆、暴击粒子。
- **背景音乐**：每章一段循环芯片音乐（用现有 Web Audio 振荡器写个极简音序器，Boss 战切换更紧张的曲子）。当前只有音效，加 BGM 是性价比最高的「炫酷」升级。
- **摄像机**：朝移动方向前瞻(look-ahead)、平滑 lerp、落地/受击微抖、Boss 入场拉镜。
- **粒子升级**：方块破碎碎块、怪物死亡「噗」烟、伤害飘字、连击数显示。
- **天气/氛围**：草原飘云已有 → 加雨、下界飘灰烬、深暗萤光孢子加密、末地紫雾。
- **代码触及**：`burst()`、主循环 hit-stop 钩子、新增 `music.js` 段落、`render()` 摄像机段。
- **工作量**：M（1 个 session 可出大半）。**风险**：低。

### v1.4 「内容扩充：怪物 × 机关 × 道具」— 加深可玩性
**目标**：把「能通关」变成「每次都有新花样」。
- **新怪物（MC 原汁原味）**：
  - 史莱姆 Slime（弹跳、被击杀分裂成小史莱姆）
  - 末影人 Enderman（瞬移、近身才攻击）
  - 烈焰人 Blaze（下界飞行、吐火球）
  - 蜘蛛 Spider（可爬墙）
  - 幻翼 Phantom（末地俯冲）
- **新机关**：
  - 活塞 Piston（推玩家/方块）、黏液块(弹跳)、灵魂沙(减速)、蓝冰(打滑)
  - 拌线/绊线钩触发、发射器升级（火球/雪球）、下落的铁砧、岩浆瀑布
- **新道具/药水**：速度、力量(剑伤翻倍)、跳跃提升、抗火(短暂免疫岩浆)、缓降、不死图腾(复活一次)。
- **代码触及**：`SOLID`/`DEADLY` 扩展为「瓦片属性表」、`spawnEnemy`/`updateEnemy` 加类型、新增 `effects[]` buff 系统。
- **工作量**：L。**风险**：中（碰撞与 buff 状态要测）。

### v1.5 「关卡与模式」— 解决「太短」
**目标**：从 4 关扩到有量、有重玩动机。
- **每章 2–3 关**：每章前置普通关 + 最终 Boss 关（共 ~10 关）。
- **新模式**：
  - **Boss Rush**：连续挑战 4 Boss，计时。
  - **无尽生存**：一波波刷怪，看能撑几波 / 多少分。
  - **每日挑战**：按日期做种子随机关卡，本地排行。
- **关卡内容**：隐藏房间、收集型目标（集齐本关绿宝石）、检查点（床）、跳跳乐段落。
- **代码触及**：`LEVELS` 扩容、新增 `MODES` 与状态机分支、种子随机 `mulberry32`。
- **工作量**：L。**风险**：中（关卡设计耗时，需大量手感测试）。

### v1.6 「视听 2.0：光影与动效」— 拉满炫酷
- **深暗之域真·黑暗**：径向光照遮罩，只照亮玩家周围 + 火把/萤石，监守者越近越暗。
- **昼夜循环**（草原）：颜色渐变 + 夜晚刷幻翼。
- **角色/怪物补帧动画**：行走、攻击、受击、死亡各 2–3 帧矢量姿态。
- **过场转场**：关卡间「方块拼合」转场动画、Boss 击败镜头特写。
- **工作量**：M–L。**风险**：低–中（光照遮罩在低端机要测性能）。

### v2.0 「系统化与可安装」— 从游戏到产品
- **成就系统**：无伤击败某 Boss、集齐绿宝石、速通、不死通关等，结算页展示。
- **皮肤/解锁**：Steve / Alex / 僵尸皮 / 骷髅皮，用绿宝石或成就解锁。
- **难度选项**：简单 / 普通 / 困难 / 极限(1 命)。
- **PWA 可安装** ⭐：加 `manifest.json` + Service Worker → 手机「添加到主屏幕」像 App 一样全屏、离线、有图标。**这是移动端体验的最大跃迁**，且与「单文件离线」目标天然契合。
- **手柄支持**：Gamepad API，接手柄即玩。
- **存档升级**：加 `saveVersion` 字段 + 迁移逻辑。
- **工作量**：L。**风险**：中（PWA 缓存策略、成就埋点要全盘梳理）。

### v2.x 远期 / 大赌注（按兴趣选做）
- **架构重构**：`game.js`(1527 行) 拆成 ES 模块（`input/audio/physics/entities/boss/render/levels`），`build.mjs` 负责打包回单文件——可维护性大升，但纯内部收益。
- **关卡编辑器**：浏览器内画地图、导出字符串塞回 `LEVELS`，甚至分享关卡码。
- **在线排行榜**：极简后端（Cloudflare Workers + KV）存每日挑战分数（需注意防作弊/隐私）。
- **对象池**：粒子/投射物复用，降 GC 抖动（v1.4 内容变多后再做）。

## 🔧 横切关注点（贯穿各版本）
- **性能**：方块绘制缓存到离屏 canvas；粒子/投射物对象池；移动端降特效档（设置里加「画质」）。
- **测试**：每加一个系统，给 `tests/smoke.mjs` 加一条断言（怪物生成/buff 生效/模式切换不崩）。
- **可访问性**：① 「减少闪光」开关（光敏癫痫安全，**应尽早做**）；② 键位重映射；③ 色盲友好配色。
- **防回归**：每个版本走 `npm run build` 重新内联 + 跑冒烟测试 + 更新 CHANGELOG。

## ✅ 优先级建议（如果让我排）
1. **先做 v1.3（打击感+BGM）** — 改动小、最快让人「哇」，直接服务「炫酷」初衷。
2. **再做 v1.4 部分内容**（史莱姆+活塞+1～2 个药水）+ **可访问性「减少闪光」开关**。
3. **然后 v2.0 的 PWA 可安装** — 一次性把手机体验提到 App 级，和你现在手机上玩的场景最契合。
4. v1.5 多关卡 / v1.6 光影按兴趣穿插。

> 一句话：**先「让现有的更爽」（v1.3），再「让它更耐玩」（v1.4/1.5），最后「让它更像个 App」（v2.0 PWA）。**

## 📏 怎么算成功（度量）
- 单局时长从 ~5 分钟 → 15+ 分钟（内容量）。
- 中端手机稳定 60fps（性能）。
- 有人愿意为「集齐绿宝石 / 刷新最高分 / 解锁皮肤」重开一局（耐玩度）。
